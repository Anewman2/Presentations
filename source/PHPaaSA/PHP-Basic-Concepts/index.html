Parsing

Reading & Interpreting of a program

PHP is an interpreted language, not a compiled one.

  • Takes input (file or stream) containing text and/or PHP instructions
  • Creates output (stream), most commonly HTML

Since PHP 4: The PHP parser interprets the input to produce opcodes for processing by the Zend Engine;
this is called preprocessing.

(0) Interpreted not compiled, (1) input text/PHP instructions, (2) output often HTML. (3) PHP parser interprets input and produces opcodes, for Zend Engine; (4) this is preprocessing.

Compiled vs. Interpreted

Programming Work-flow

Compiled
  • Write human-readable source code
  • Use a compiler to Translate source code and generate machine code
  • Execute machine code for debugging
  • Repeat…
Interpreted
  • Write human-readable source code
  • Execute source code for debugging with an interpreter
  • Repeat…
(0) In a compiled work-flow; (1) write code, (2) compile, (3) execute, (4) repeat. (5) In an interpreted work-flow; (6) write code, (7) execute, (8) repeat.

Preprocessing

A PHP script: from human-readable code
to an executing program

  1. Preprocessor
    1. Lexical Analyzer (lexer)
      1. Reads in the human-readable code
      2. Interprets the code into machine-digestible tokens
      3. Passes the tokens to the Parser
    2. Parser
      1. Creates an operation code array (opcode / op array) from the tokens
        – The op array is also known as an abstract syntax tree (AST)
      2. Passes the op array to the Executor
  2. Executor
    1. Steps through the op array
    2. Executing each opcode in the Zend Engine
(0) A PHP script first gets consumed by the (1) preprocessor, which consists of a (2) lexer and a (3) parser. The lexer (4) reads the code, (5) interprets to tokens, (6) then passes the tokens to the parser. The parser (7) creates an op array (8) AKA AST and (9) passes the op array to the executer. Then the interpreted code is passed to the (10) executor, which (11) iterates the op array and (12) executes the opcode.

The Zend Engine (v2.0)

The Virtual Machine

The Zend Engine simulates a physical computer; the VM architecture provides flexibility and portability. While a standard (x86) computer processor has 75 base operations, the Zend Engine has approximately 150 instruction (opcodes).

  • Keeps track of requests
  • Processes scripting files
  • Handles variables and resources

The opcode set includes typical logic and math operations, as well as complex operations such as include(). A VM is always slower than the host machine; extra speed is obtained by performing complex instructions as a single VM operation.

The Zend Engine (0) simulates a computer, flexibility & portability. (1) Processor has 75 ops, Zend Engine has ~150 ops. Zend Engine (2) tracks requests, (3) processes scripts, (4) handles variables & resources. (5) opcode set includes logic & math ops, as well as complex ops like `include()`. (6) VM is slower than host, extra speed obtained by complex instruction as single op

PHP Tags

Opening and Closing tags

  1. <?php and ?> Always Available
  2. <script language="php"> and </script> Always Available
  3. <? and ?> Short Style
  4. <?= and ?> Echo - Short Style
  5. <% and %> ASP Style
  6. <%= and %> Echo - ASP Style

The opening tag tells the interpreter to turn on

The closing tag tells the interpreter to turn off

  • Tag Sets 1 and 2 are always available
    • Other tag sets should be avoided when developing for redistribution
    • Set 1 is most common and recommended
  • Tag Sets 3 and 4 are Short style – Requires a configuration change
    • Configuration file (php.ini) – short_open_tag
    • Compiled with --enable-short-tags
    • Since ver 5.4 parser recognizes short tags regardless of configuration
  • Tag Sets 5 and 6 are ASP style – Requires a configuration change
    • Configuration file (php.ini) – asp_tags
  • Since ver 5.3 parser allows an opening tag (minimum of one white space character) by itself in a file
  • A Close tag at the end of the file is optional
    • Helpful so unwanted white space does not occur
